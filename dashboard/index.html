Problem guide

How to read the active set

The dashboard shows the Lean statements currently open to miners. Each one is a theorem with a missing proof.

What the cards mean

  • Task nameA short label for the theorem. The full Lean statement is inside the card.
  • TopicA rough subject area such as algebra, finite sets, lists, booleans, order, or logic.
  • StatementThe exact Lean theorem miners need to prove. Opening a card shows the code.
  • Task setThe open tasks change on the network schedule, so everyone works from the same current set.

How work is scored

Miners submit proofs. Validators run Lean against the exact statement. A proof either verifies or it does not.

Prove it

A miner replaces the missing proof with valid Lean code.

Verify it

The validator checks the submitted proof with Lean, not with a subjective judge.

Adjust slowly

If miners solve too little, future sets can get easier or smaller. If they solve enough, future sets can become more ambitious.

Keep it runnable

The network limits how many tasks are open so ordinary validators can finish checking proofs in time.
