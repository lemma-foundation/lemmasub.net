Testnet problem guide

Current testnet problem set

Testnet now runs the production problem-selection rules against the testnet task supply. This dashboard shows the active set miners are working on right now.

Current testnet families

  • LogicTrue, False, implication, negation, and direct proof-from-assumption statements.
  • And / orBuilding both sides, choosing one side, or unpacking assumptions.
  • EqualityReflexivity, symmetry, and short equality chains over natural numbers.
  • Natural numbersZero, successors, addition, order, nonzero facts, and small identities.
  • ListsEmpty lists, list length, and append identities.
  • BooleansSmall true/false facts such as negating a boolean value.
  • ExistenceFind an example and prove that it has the requested property.

Difficulty meaning

The current testnet supply may still show mostly Easy problems, but the difficulty bands use the same meaning as production.

Easy

One direct idea: simplify, introduce, split, or use a standard fact.

Medium

A short sequence of proof steps or a few facts in the right order.

Hard

More structure, more search, or a less obvious proof path.

Frontier

The hardest active edge when easier patterns are being solved reliably.

Mainnet problem guide

Mainnet problems are generated

On mainnet, paid problems are procedurally generated. The active set is created automatically from approved mathematical sources in a repeatable way, then checked before it appears here.

What mainnet uses

  • All paid problems are generatedNo one hand-picks the active paid set. The generator creates exact Lean statements for miners to prove.
  • Built from approved sourcesThe generator starts from pinned mathematical material, so the same public inputs produce the same problem set.
  • Checked before activationA problem must be valid Lean, safe to publish, and not already solved by the basic checker.
  • Practice sets stay separateTestnet, benchmark, and curriculum problems can exist, but they are not the paid mainnet problem set.

Difficulty bands

Difficulty is a plain signal for how much proof work a miner should expect.

Easy

A direct proof. Usually one obvious idea or standard simplification.

Medium

A short proof with a few connected steps or facts.

Hard

Needs planning, search, or less obvious theorem knowledge.

Frontier

The edge of the current set. Expect experimentation before a proof lands.
