Verified reasoning data

Miners submit proofs. Lemma verifies them.

Lemma is a way to build training data that does not rely on a guess. A miner sends in a proof, a validator runs a verifier, and a passing proof becomes a public example that models can learn from.

Proof flow

  1. Problem Show a task

    A theorem or reasoning problem is posted.

  2. Proof Submit an answer

    A miner sends proof text that should solve it.

  3. Verifier Run the verifier

    Software decides pass or fail.

  4. Dataset Keep what passed

    Accepted proofs become open training examples.

What it is

A filter for useful reasoning work.

A lot of AI output sounds right. That is not the same as being right. Lemma focuses on work that can be verified by software.

The simple rule is: if the proof passes, it can be kept. If it fails, it does not become training data.

How it works

Four steps, no mystery layer.

Miners can use any method they want to find proofs. Validators only care whether the final proof passes the verifier.

  1. 1 A task is posted

    It says what needs to be proved.

  2. 2 A miner submits a proof

    The proof is the thing being judged.

  3. 3 A validator verifies it

    The verifier gives a clean pass or fail.

  4. 4 The good proof is saved

    It becomes part of an open dataset.
