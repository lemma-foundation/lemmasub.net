Lean proofs on Bittensor

Lemma

A proof either checks or it does not. Miners submit Lean files, validators run the pinned checker, and rewards follow verified results. If multiple miners verify in the same validator batch, the batch reward is split equally.

Protocol

Small surface. Hard problem.

Lemma keeps the live protocol narrow so miners can focus on the real work: turning theorem targets into Lean-verified proofs.

01

Read the target

Everyone works against the same theorem statement, imports, and pinned Mathlib revision.

02

Produce a proof

Use any out-of-band method: a proving model, local search, manual work, or a mix.

03

Submit and serve

The CLI verifies the Lean file, stores it locally, and can serve it to validators.

04

Verify and reward

Validators run Lean. Passing proofs receive the WTA signal, with same-batch ties split equally.

Proof object

The submitted file is the work.

Informal reasoning can help people understand a proof, but it is not the reward object. The live system checks the formal Lean file against the target theorem. No prose score, no reasoning score, no proof efficiency bonus. 

import Mathlib

namespace Submission

theorem nat_two_plus_two_eq_four : (2 : Nat) + 2 = 4 := by
  rfl

end Submission

Miners

Generate proofs your way.

The subnet does not prescribe how a proof is found. It only asks for the finished Lean file and a proof-serving miner when you want to participate live.

Miner guide

Validators

Run the checker, publish the signal.

Validators poll miners, reject unfinished or unsafe proof files, and write weights from objective verification results.

Validator notes

Direction

Grow proof capacity.

The public protocol stays simple while the target supply and miner tooling improve. The useful frontier is not prettier explanations; it is more reliable Lean proof production.

Binary checking

Lean accepts the proof or it does not. Shortcuts like sorry and new axioms are rejected.

Equal tie split

When multiple valid proofs land in the same validator batch, the reward is split evenly among them.
