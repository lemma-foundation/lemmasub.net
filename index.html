How it works

AI can guess. Lean can check.

Miners search for proofs. Validators run the checker. Correct proofs become training data. The reward is tied to the verified artifact, not to prose or claimed effort.

01

Task

Lemma publishes exact Lean theorem tasks.

02

Search

Miners use models, tactics, retrieval, or human insight.

03

Submit

The artifact is a Lean proof for the exact task.

04

Verify

Validators run the pinned Lean environment.

05

Reward

Accepted proofs receive normal Bittensor miner credit.

06

Train

The proof enters the open corpus for future provers.

Public dataset

The corpus is the product.

Every accepted proof becomes a replayable dataset row: theorem statement, imports, toolchain, proof, hashes, verifier result, and solver attribution.

A lemma is a stepping stone to a larger proof. Lemma creates many of those stepping stones for mathematical AI.

Frontier benchmarks

Hard problems measure progress.

Google DeepMind's Formal Conjectures, lean-eval, miniF2F, PutnamBench, and the IMO Grand Challenge are benchmarks for the provers trained on Lemma data. They are not the v1 reward stream.

Formal Conjectures

Open Lean conjectures from Google DeepMind's public repository.

lean-eval

A leaderboard of hard Lean formalization problems.

PutnamBench

Formal theorem-proving benchmark based on Putnam problems.

Use Lemma

Simple commands, exact proofs.

The CLI should make tasks, verification, submissions, and corpus replay easy to inspect.

uv sync
uv run lemma setup
uv run lemma tasks list
uv run lemma tasks inspect <task-id>
uv run lemma verify <task-id> --submission Submission.lean
uv run lemma corpus replay corpus.jsonl

Lemma is independent. Google DeepMind's Formal Conjectures is referenced as a public benchmark; no endorsement is implied.
