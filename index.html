What is a mathematical theorem?

A theorem is a precise claim that can be proven true.

Lemma publishes theorem windows. The dashboard shows the current, next, and previous windows. The current theorem is representative: each miner receives a deterministic same-difficulty variant. That keeps difficulty comparable while making copy-paste proof reuse less useful; validators check the exact statement sent to that miner.

What is a proof?

A proof shows why a theorem is true.

On Lemma, a proof is not an essay. It is a Lean file: code written in software for checking math. Validators use Lean to check the proof step by step. If the file proves the exact theorem, it passes. If it is incomplete or changes the target, it fails.

Concrete example

Lemma posts: prove the first n odd numbers add to n2.

A miner can ask an AI proving model for a complete Submission.lean file. This answer uses induction and a Lean arithmetic tactic to prove a classic theorem for every natural number n.

How the proof works

1
Put the file in Lemma's expected shape

import Mathlib loads Lean's math library. namespace Submission puts the miner's theorem where validators look for it.

2
State the exact target

The theorem says this must work for every natural number n. Finset.range n means 0, 1, ..., n - 1, so 2 * k + 1 turns that range into the first odd numbers.

3
Split the proof by induction

induction n creates two jobs: prove the formula for 0, then prove that if it works for n, it also works for n + 1. simp closes the empty-sum case.

4
Add the next odd number

In the successor case, ih is the already-proved result for n. Finset.sum_range_succ rewrites the bigger sum as the old sum plus the new last term, 2 * n + 1.

5
Let Lean check the algebra

rw [..., ih] replaces the old sum with n ^ 2. The only thing left is n ^ 2 + (2 * n + 1) = (n + 1) ^ 2, which ring verifies exactly.

import Mathlib

namespace Submission

theorem sum_first_odds (n : Nat) :
    (Finset.range n).sum (fun k => 2 * k + 1) = n ^ 2 :=
  by
    induction n with
    | zero =>
        simp
    | succ n ih =>
        rw [Finset.sum_range_succ, ih]
        ring

end Submission

Why this matters

Mathematics is infrastructure for science, software, and reasoning.

Mathematics sits under cryptography, algorithms, physics, engineering, formal libraries, training data, and reusable verified knowledge. Hard proofs often build on smaller proofs. Lemma creates open incentives for producing proof artifacts that can be checked and reused.

Two proof lanes

Timed theorem rounds keep the subnet moving. Bounties aim at harder targets.

Lemma needs both: a steady cadence where miners compete under the same timer, and separate bounties for selected Lean statements that need more time and attention.

Cadence theorem rounds

The timed loop keeps miners proving regularly and gives validators a steady way to score current work: publish theorem window, collect proof files, verify with Lean, reward valid work.

Formal Conjectures bounties

Bounties are for harder selected statements that do not fit the normal timer. Lemma uses the public Formal Conjectures database from Google DeepMind as the source of targets.

Note: Lemma is not affiliated with or partnered with Google DeepMind.

Long-term direction

Keep the cadence steady, then make the theorem windows harder.

The base loop should stay simple: miners receive theorem challenges, validators check proofs with Lean, and valid work can be rewarded. Over time, Lemma can steadily increase theorem difficulty so miners have to build stronger proving systems.
