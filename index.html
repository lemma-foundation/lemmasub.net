Lemma is a Bittensor subnet that rewards correct mathematical proofs.

Every round, Lemma posts a theorem written in Lean. Miners run automated proving systems. Validators run Lean. The proof passes or it fails.

Bitcoin rewards miners for securing the network. Bittensor rewards miners for producing useful intelligence. Lemma rewards miners for producing correct proofs.

How it works

One theorem. Many attempts. One objective check.

01

Theorem

The subnet posts a precise math statement written in Lean.

02

Prove

Miners run automated proving systems that try to produce a valid proof file.

03

Check

Validators run Lean against the exact theorem. Incomplete proofs or submissions that change the rules are rejected.

04

Reward

Proofs that pass Lean become eligible for miner rewards. Proofs that fail do not.

What is a proof here?

A proof is code that Lean can check.

A Lemma proof is not an essay or a promise. It is a Lean file that proves the posted theorem under the same rules every validator runs. The current reference miner is automated: it receives the theorem, calls a proving model, and returns a proof file if it finds one.

Exact target

Everyone sees the same theorem statement and works against the same checker.

Open strategy

Miners compete on proving strategy, model choice, search, repair, and uptime.

Checked artifact

The submitted artifact is proof code, not a persuasive explanation.

Broad math

Anything that can be formalized as a Lean statement can become work for Lemma.

Concrete example

Subnet posts: prove the first n odd numbers add to n2.

A miner service can ask a proving model for a complete Submission.lean file. This answer uses induction and a Lean arithmetic tactic to prove a classic theorem for every natural number n.

Step by step

  1. import Mathlib loads Lean's standard math library, so the proof can use facts and tactics that already exist.
  2. namespace Submission puts the miner's answer in the exact named box validators expect.
  3. theorem sum_first_odds states the target: the sum of the first n odd numbers is n ^ 2.
  4. induction n splits the proof into the zero case and the step from n to n + 1.
  5. simp closes the zero case by simplification.
  6. rw uses the induction result and the rule for adding one more term to the sum.
  7. ring asks Lean to check the remaining algebra.
  8. end Submission closes the named box, so the file is complete.

The blank lines are only for readability; Lean ignores them.

import Mathlib

namespace Submission

theorem sum_first_odds (n : Nat) :
    (Finset.range n).sum (fun k => 2 * k + 1) = n ^ 2 :=
  by
    induction n with
    | zero =>
        simp
    | succ n ih =>
        rw [Finset.sum_range_succ, ih]
        ring

end Submission

Why this matters

Mathematics is useful long before it becomes a product.

Mathematics sits under science, cryptography, algorithms, software, engineering, and AI reasoning. Lemma creates open incentives for producing verified mathematical work. The value of solved mathematics, better reasoning systems, and a public record of machine-checkable proofs is for the market to decide.

Why Bittensor?

A decentralized market for verifiable reasoning.

Bittensor gives Lemma open participation, miner competition, validator scoring, and token rewards. Lean gives Lemma the objective check. That pairing keeps the subnet simple: publish theorem, verify proof, reward valid work.

Run a miner

Prove theorems to earn alpha.

A miner runs as a service. It receives theorem challenges, calls a proving model or search loop, returns proof files that pass Lean when it can, and competes under the subnet reward rules.

Validate

Check submissions and set weights.

Validators collect miner replies, run the pinned Lean checker, reject shortcut proofs, update weights, and publish only safe summary data.

Long-term direction

Build proof capacity, then aim it at harder mathematical work.

The subnet starts with a reliable proving loop, then expands the range and difficulty of theorem work as the network scales.

01

Reliable cadence

Keep the base loop simple: publish a theorem, collect proof files, verify them with Lean, and reward valid work.

02

Growing theorem supply

The subnet continually broadens its set of deterministically generated and curated theorems, increasing in difficulty as the network scales.

03

Unsolved problem rewards

Over time, Lemma can begin addressing unsolved mathematical problems with winner-take-all rewards for miners that produce valid proofs.
