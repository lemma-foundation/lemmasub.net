Bittensor subnet for mathematical AI

Train the best open mathematical prover.

Lemma is a Bittensor subnet where miners create Lean-verified theorem/proof data. Validators check every proof with the Lean kernel. The resulting open corpus trains stronger AI provers, which can be tested against frontier benchmarks like Google DeepMind's Formal Conjectures.

AI can guess. Lean can check. Lemma pays for the checked data.

GitHub How it works

The loop

Task -> Proof -> Lean check -> Reward -> Corpus

Miners search for proofs. Validators run the Lean checker. Correct proofs receive normal Bittensor miner credit and become rows in an open corpus.

  1. TaskLemma publishes exact Lean theorem targets.
  2. ProofMiners use any prover stack they want.
  3. CheckValidators accept only proofs that build in Lean.
  4. DataAccepted proofs become training examples.

Why Lean

Lean turns guesses into checked data.

A model can produce a proof that looks plausible. Lean checks whether it actually follows from the theorem statement and allowed imports. Lemma uses that pass/fail boundary as the reward signal.

Corpus

The corpus is the product.

Each row records the theorem statement, imports, toolchain, proof, hashes, verifier result, source metadata, and attribution needed to replay the check later.

{
  "task_id": "lemma.sample.true_intro",
  "proof_sha256": "14ae...",
  "source_license": "CC-BY-4.0",
  "verification": { "passed": true }
}

Mining

Miners produce checked proof artifacts.

Miners can use local models, hosted APIs, tactics, retrieval, search, or human-written proofs. Lemma only cares whether the final `Submission.lean` passes the pinned verifier.

uv run lemma setup
uv run lemma mine --once

Validation

Validators check, score, and publish.

Validators reject invalid targets, duplicate proof identities, and failed Lean builds. They score the first accepted unique proof for each task and publish corpus rows after the scoring window.

uv run lemma worker --check
uv run lemma validate --once --no-set-weights

Benchmarks

Frontier problems measure progress.

Google DeepMind Formal Conjectures, lean-eval, miniF2F, PutnamBench, and the IMO Grand Challenge are benchmarks for provers trained on Lemma data. They are evaluation targets, not the v1 reward path.

Get started

Run the proof-data loop locally.

Start by listing active tasks, checking one theorem, and running one miner or validator iteration.

uv sync --extra btcli
uv run lemma setup
uv run lemma tasks list
uv run lemma task show lemma.sample.true_intro

Lemma is independent. Google DeepMind Formal Conjectures is referenced as a public benchmark; no endorsement is implied.
