Lemma

Lemma publishes formal proof targets and rewards miners who produce Lean-verified proofs. Correctness is checked by Lean: pass or fail.

What miners work on

Published formal targets, not subjective prompts.

A Lemma task is a precise Lean problem with a statement, target hash, toolchain, and submission policy. Miners write the proof; Lean decides whether it is accepted.

Exact statement

The target fixes the theorem, imports, namespace, and allowed submission shape.

Lean check

The same verifier path checks every submitted Submission.lean file.

Reward handling

The CLI prepares the proof and payout package while custody details stay secondary.

How proof verification works

Proof correctness is binary.

The live reward path cares about one question: does Lean accept the proof for the exact published statement under the published policy? If yes, it passes. If not, it fails. 

import Mathlib

namespace Submission

theorem target : True := by
  trivial

end Submission

Use the repo
