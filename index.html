Verifier-grounded data subnet

Verifier-grounded training data.

Lemma is a Bittensor subnet where miners produce artifacts accepted by deterministic verifiers, and validators publish those accepted artifacts as open training corpora. Lean theorem proving is the first production domain.

AI can guess. Verifiers can check. Lemma pays for checked data.

GitHub How it works

The mechanism

Tasks, artifacts, deterministic checks, public rows.

Lemma turns artifact search into public training data: publish tasks, search for accepted artifacts, check them with pinned verifiers, reward verified units, publish corpus rows, and use that data to train stronger reasoning models.

  1. TasksValidators expose exact verifier targets.
  2. ArtifactsMiners use any search stack they want.
  3. ChecksThe verifier decides pass or fail.
  4. DataAccepted artifacts become public rows.

Lean first

Math is the wedge. Verified data is the product.

Lean is the first clean instance because its verifier is deterministic, mature, and backed by Mathlib. A model can write something that looks like a proof. Lean checks whether it really follows from the theorem statement and allowed imports.

Corpus

The corpus is the product.

Each accepted row records the domain, verifier, prompt, accepted artifact, hashes, attribution, source metadata, reward status, and replay information. The current public corpus is the Lean domain corpus.

Model miners

Lemma data feeds model competition.

Affine-style model miners can train on Lemma's public corpora. Affine rewards model dominance; Lemma produces the verifier-grounded corpora that help reasoning models improve.

uv run lemma export-corpus \
  --domain lean \
  --format jsonl \
  --out data/lean_corpus.jsonl

Mechanism class

Any deterministic verifier can become a domain.

Lean remains the only production domain today. The adapter architecture is built for future domains such as Verus/Rust verification, SAT/SMT certificates, optimization certificates, and cryptanalysis witnesses.

  1. LeanTheorem-proof pairs.
  2. VerusRust code, specs, proof annotations.
  3. SAT/SMTFormulas and certificates.
  4. OptimizationPrimal and dual certificates.

Validation

Validators check artifacts, not claims.

Validators enforce task binding, dispatch to the domain verifier, reject duplicates, score the first accepted unique artifact for each active slot, and publish replayable corpus rows. Unsolved-slot value is not redistributed to current solvers.

uv run lemma worker --check
uv run lemma validate --once --no-set-weights

Benchmarks

Benchmarks measure progress. They are not the product.

The corpus trains models. Benchmarks show whether those models improve. Formal Conjectures, lean-eval, miniF2F, PutnamBench, and the IMO Grand Challenge are evaluation targets, not the v1 reward path.

Research context

AlphaProof-style ideas, open corpus mechanics.

AlphaProof-style systems showed the power of large formal curricula and verifier-grounded feedback. Lemma takes the data flywheel open: a market for verified training data, not a closed lab dataset.

  • AlphaProof and AlphaGeometry research
  • LeanNavigator state-graph generation
  • LeanConjecturer
  • LeanDojo

Lemma is independent. AlphaProof, Google DeepMind Formal Conjectures, miniF2F, PutnamBench, LeanDojo, LeanNavigator, Affine, Verus, and the IMO Grand Challenge are referenced as research context, integration context, or evaluation targets only. No endorsement is implied.
