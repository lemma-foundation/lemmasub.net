Lemma rewards proofs Lean can verify.

Lemma is a Bittensor subnet for machine-checkable mathematics. Miners produce Lean proof files, validators run the pinned checker, and rewards follow proof correctness.

Subnet loop

One target, one proof file, one mechanical check.

  1. Theorem

    The subnet publishes a precise Lean statement, imports, and target environment.

  2. Proof

    Miners work however they want, then submit a complete Submission.lean file.

  3. Verify

    Validators replay the file with Lean. The proof passes or fails.

  4. Reward

    Current verified cadence work earns automatic subnet rewards. Bounties pay manual campaign rewards.

Two task surfaces

Cadence is automatic. Bounties are focused campaigns.

Cadence tasks are recurring targets for registered miners. Formal Conjectures bounties are manual owner-emission campaigns for larger theorem work and do not require subnet registration.

Concrete example

A submitted proof is just a Lean file.

Suppose the target is the classic claim that the first n odd numbers add to n2. A miner can use a proving model, local search, manual work, or all three, but the reward-critical object is the file Lean accepts.

  1. import Mathlib loads Lean's standard math library.
  2. namespace Submission puts the answer where validators expect it.
  3. theorem sum_first_odds states the exact target.
  4. induction n splits the proof into the zero case and the successor case.
  5. simp closes the zero case by simplification.
  6. rw uses the induction result and the rule for adding one more term.
  7. ring asks Lean to check the remaining algebra.
import Mathlib

namespace Submission

theorem sum_first_odds (n : Nat) :
    (Finset.range n).sum (fun k => 2 * k + 1) = n ^ 2 := by
  induction n with
  | zero =>
      simp
  | succ n ih =>
      rw [Finset.sum_range_succ, ih]
      ring

end Submission
