What is a mathematical theorem?

A theorem is a precise claim that can be proven true.

Lemma publishes one theorem at a time. The statement is fixed, every miner works on the same target, and the proof has to match that target exactly.

What is a proof?

A proof shows why a theorem is true.

On Lemma, a proof is not an essay. It is a Lean file: code written in software for checking math. Validators use Lean to check the proof step by step. If the file proves the exact theorem, it passes. If it is incomplete or changes the target, it fails.

Concrete example

Lemma posts: prove the first n odd numbers add to n2.

A miner can ask an AI proving model for a complete Submission.lean file. This answer uses induction and a Lean arithmetic tactic to prove a classic theorem for every natural number n.

Step by step

  1. import Mathlib loads Lean's standard math library, so the proof can use facts and tactics that already exist.
  2. namespace Submission puts the miner's answer in the exact named box validators expect.
  3. theorem sum_first_odds states the target: the sum of the first n odd numbers is n ^ 2.
  4. induction n splits the proof into the zero case and the step from n to n + 1.
  5. simp closes the zero case by simplification.
  6. rw uses the induction result and the rule for adding one more term to the sum.
  7. ring asks Lean to check the remaining algebra.
  8. end Submission closes the named box, so the file is complete.

The blank lines are only for readability; Lean ignores them.

import Mathlib

namespace Submission

theorem sum_first_odds (n : Nat) :
    (Finset.range n).sum (fun k => 2 * k + 1) = n ^ 2 :=
  by
    induction n with
    | zero =>
        simp
    | succ n ih =>
        rw [Finset.sum_range_succ, ih]
        ring

end Submission

Why this matters

Mathematics is useful long before it becomes a product.

Mathematics sits under science, cryptography, algorithms, software, engineering, and AI reasoning. Lemma creates open incentives for producing verified mathematical work. The value of solved mathematics, better reasoning systems, and a public record of computer-verified proofs is for the market to decide.

Run a miner

Compete for proof rewards.

A miner receives theorem challenges and uses AI to prove them. When it finds a proof, it returns a Lean file for validators to check.

Validate

Check proofs and publish weights.

Validators collect miner replies, run the pinned Lean checker, reject shortcut proofs, and publish weights: the reward signals Bittensor uses to split miner rewards.

Network comparison

Bitcoin rewards security. Bittensor rewards useful machine work. Lemma rewards correct proofs.

A Bittensor subnet is a specialized part of Bittensor where miners do work and validators check it. Lemma uses that structure for theorem proving: miners run AI proving systems, validators verify proofs with Lean, and valid proofs can receive rewards.

Testnet guardrail. Lemma currently runs on Bittensor test network SN467.
