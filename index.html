Google DeepMind Formal Conjectures -> Lemma bounties

A bounty market for solving Google DeepMind's Formal Conjectures.

Lemma is a Bittensor subnet where miners compete to prove open mathematical statements from Google DeepMind's public formal-conjectures repository. Validators run Lean to check each proof. When a theorem is solved, Lemma publishes the proof artifact and prepares an upstream pull request.

The loop

From open conjecture to public proof.

The homepage version is simple: Lemma chooses an open Lean theorem, opens a bounty, checks submitted proofs with Lean, and sends solved work back toward the public corpus.

02

Open a bounty

The theorem becomes the active proof challenge. If funded, the bounty can grow while it remains unsolved.

03

Submit a proof

A miner publishes a Lean proof for the exact theorem.

04

Check with Lean

Validators run the same verifier. The proof either builds or it does not.

05

Attest

Validators attest when the exact theorem checks.

06

Claim the bounty

If enough validator stake attests to the proof, the solver can claim the bounty.

07

Publish the proof

Lemma writes a public proof artifact with the accepted Lean file and verification record.

08

Send it upstream

Lemma prepares a pull request back to Formal Conjectures so maintainers can review the solved theorem.

Source corpus

What is Google DeepMind's Formal Conjectures?

Formal Conjectures is a public Google DeepMind repository of mathematical conjectures written in Lean. Many entries are open research problems: the statement has been formalized, but the proof is not yet known in Lean.

Lemma turns those open statements into economic challenges. Instead of asking whether an AI answer sounds right, Lemma asks whether Lean accepts the proof.

Lemma is independent and is not endorsed by Google DeepMind. It uses the public google-deepmind/formal-conjectures repository as target material.

Bittensor roles

Proof searchers, proof checkers, and Lean.

Miners

Miners are proof searchers.

They use AI systems, tactics, scripts, or human insight to find a Lean proof.

Validators

Validators are proof checkers.

They run the same Lean environment and attest only when the proof builds.

Lean

Lean is the judge.

No essays, no subjective scoring, no vibes. The proof checks or it fails.

After a theorem is solved

Solved proofs should flow back upstream.

A solved theorem becomes a public proof artifact: the accepted Lean proof, the theorem it proves, and the verification record. For Formal Conjectures targets, Lemma can prepare a pull request that marks the statement solved and links to the artifact.

Upstream maintainers still review the pull request. A merged PR is the public publication path, not the rule for whether a solver can claim a Lemma bounty.

Lemma turns open Google DeepMind Formal Conjectures into Lean proof bounties and publishes the solved work.

Lemma is independent and is not endorsed by Google DeepMind or the Formal Conjectures authors. Lean verification checks the published formal statement; it does not guarantee that an informal mathematical conjecture was perfectly captured. Upstream pull request acceptance is independent from Lemma bounty eligibility.
