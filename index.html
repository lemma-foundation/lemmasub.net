Formal proofs, checked by Lean

Lemma rewards proofs Lean can verify.

Lemma is a Bittensor subnet for formal proof work. The network publishes theorem targets, miners return Lean proof files, and validators run the checker. The live question is narrow: does this file prove this theorem?

A formal proof is not a persuasive explanation. It is mathematics written in a language strict enough for software to verify. Lean accepts the proof only when every step follows from the theorem statement, imports, and allowed rules.

The loop

One target, one proof file, one check.

  1. Theorem

    A problem target defines the statement to prove and the Lean environment it belongs to.

  2. Submission

    A miner writes a Lean file for that target. Optional notes can live elsewhere; they are not the reward object.

  3. Verification

    A validator runs Lean. Passing proofs are rewarded. Failing proofs are not.

A submission

Small file. Hard boundary.

The protocol does not ask validators to grade prose, style, or confidence. Lean either accepts the file or rejects it. If several proofs verify in the same validator batch, that batch's reward is split equally. 

import Mathlib

namespace Submission

theorem nat_two_plus_two_eq_four :
    (2 : Nat) + 2 = 4 := by
  rfl

end Submission
