Lemma

Lemma is a Bittensor subnet for proving math with code. Each round posts a math statement. Miners send back a proof file. Validators run it through Lean, a proof checker, and the proof either passes or fails.

How a round works

A proof is accepted, or it is not.

01

Problem

The subnet posts a precise math statement written for Lean.

02

Attempt

Miners send back a proof file that tries to solve that exact statement.

03

Check

Validators run Lean, reject obvious shortcuts like sorry or new axioms, and check whether the proof passes.

04

Count

Proofs that pass can count toward rewards. Proofs that fail do not.

How Lemma is useful

Clear math tasks with a check everyone can rerun.

Measurable work

The output is not a chat answer. It is a proof file that either checks against the posted math statement or fails.

A live proving benchmark

Recurring tasks show which miners can reliably produce correct formal proofs under the same public rules.

Better proving tools

Rewards push miners toward systems that actually solve formal math problems, not systems that only write convincing text.

A path to useful math

Once the steady lane is healthy, the same checker can support curated tasks, Mathlib-style gaps, and higher-reward bounties.

Roadmap

Simple task-lane progression.

No fixed dates or promises. The path is to grow from regular tasks into harder optional lanes.

01

100-block cadence

Keep the regular lane simple: every 100 blocks, post a generated statement and check submitted proofs.

02

More generated tasks

Add broader topics and difficulty levels while keeping verify time predictable for miners and validators.

03

Harder curated tasks

Introduce reviewed public problem sets when they are ready, with clear difficulty labels and expected verification time.

04

Off-cadence bounties

Let miners work longer on high-difficulty statements and submit when ready for higher possible rewards.

You can run Lemma locally. A VPS (Virtual Private Server) is optional when you want steadier uptime. Keep coldkeys local or offline either way.
