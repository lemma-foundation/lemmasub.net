Verified Reasoning Network

Reasoning AI can prove, not just say.

Lemma is a network for producing verified reasoning data. Miners solve hard reasoning tasks, validators check the answers with deterministic verifiers, and accepted solutions become open training data for stronger AI.

AI can guess. Verifiers can check. Lemma pays for checked reasoning.

How it works

Miners solve. Validators verify. The network publishes checked reasoning data.

Lemma turns reasoning tasks into verified training examples. A miner searches for a proof, a validator runs the checker, and an accepted solution becomes part of the public corpus.

  1. TasksValidators expose formal reasoning targets.
  2. SolutionsMiners search for proofs using any stack they want.
  3. VerificationA deterministic verifier decides pass or fail.
  4. DataAccepted solutions become public training data.

Why verified reasoning?

Fluent answers are not enough.

Modern AI can produce answers that sound correct. Lemma focuses on answers that can be mechanically checked. A proof that passes a verifier is more useful than a claim that only sounds plausible.

Lean first

Lean is the first production checker.

In v1, Lemma uses Lean theorem proving because Lean gives the network a mature deterministic verifier. A submitted proof either passes in the pinned Lean environment or it does not.

Verified data

The product is an open library of checked reasoning.

Each accepted row records the task, proof, verifier, source, license, attribution, dependencies, and replay metadata. The current production corpus is Lean proof data.

Network roles

A market for checkable reasoning.

Lemma runs as a network on Bittensor. Miners compete to solve active tasks, validators verify and score accepted work, and model builders can train on the public corpus.

  1. MinersFind valid proofs for active tasks.
  2. ValidatorsRun deterministic checks and score accepted work.
  3. Model buildersTrain on the public corpus.
  4. Future domainsAttach new verifiers once they meet the same standards.

Roadmap

From Lean proofs to broader verified reasoning.

Lean theorem proving is the only production domain today. Future roadmap examples include Verus for Rust verification, SAT/SMT certificates, optimization certificates, and cryptanalysis witnesses.

uv run lemma export-corpus \
  --domain lean \
  --format jsonl \
  --out data/lean_corpus.jsonl
