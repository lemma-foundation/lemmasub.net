What it is

A subnet for exact answers, not persuasive prose.

1

The target is formal.

Each task is a precise Lean theorem with pinned imports and environment.

2

The work is open.

Miners can use search, models, manual proof work, or their own tooling.

3

The check is binary.

Validators run Lean. The submitted file either verifies or it does not.

Why Lean

Lean is the referee.

Lean is proof-checking software. It does not trust a paragraph that sounds right; it checks each step against formal rules.

That makes Lemma's reward path simple: the valuable object is the proof file. Informal reasoning can help humans, but the subnet pays for machine-checkable correctness.

Proof example.

One idea, written in two forms.

Suppose the task says the first n odd numbers add to n2. The explanation helps a person follow the idea. The Lean file is what validators check.

  1. import Mathlib loads the math library.
  2. namespace Submission matches the expected wrapper.
  3. theorem sum_first_odds states the exact target.
  4. induction n proves zero, then the next number.
  5. simp, rw, and ring close the checked steps.
import Mathlib

namespace Submission

theorem sum_first_odds (n : Nat) :
    (Finset.range n).sum (fun k => 2 * k + 1) = n ^ 2 := by
  induction n with
  | zero =>
      simp
  | succ n ih =>
      rw [Finset.sum_range_succ, ih]
      ring

end Submission

How it works

Two kinds of work, one verification rule.

Cadence

Recurring subnet tasks.

Registered miners usually use prover APIs, local tools, or manual work to find a Lean proof. Validators only score the file that passes.

Bounty

One focused campaign.

Open to anyone. One manual winner-take-all task appears with a listed reward and a directly linked source.

Bounty source note: Lemma is not affiliated with Google DeepMind. Bounty tasks use Google DeepMind Formal Conjectures material under its published Apache 2.0 and CC-BY 4.0 licensing, with the original source linked.

Scoring

Cadence scoring and bounty rewards are separate.

Cadence scoring

Validators ask registered miners for the current proof after reveal. Difficulty-weighted rolling scores normalize into validator weights.

Cadence timing

Each theorem window is 100 blocks, about 20 minutes at a 12-second block pace.

Bounty rewards

Bounties are outside automatic validator weights. The dashboard shows the one current campaign, source, status, and reward.
