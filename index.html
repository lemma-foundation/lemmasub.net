Tiny example

A Lean submission is checked exactly.

This toy theorem shows the basic artifact: a formal statement, proof code, and a verifier result. In competition, that is only the first gate; the submission still has to match an active task, pass policy checks, be unique, and win its slot.

Statement theorem add_zero_example (n : Nat) : n + 0 = n := by
Proof exact Nat.add_zero n
Pass

Lean accepts this proof script for this statement. A competition submission would still need the right task id, task version, target hash, allowed imports, no `sorry` or `admit`, a unique proof identity, and the winning rank under validator scoring.
