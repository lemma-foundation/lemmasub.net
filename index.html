Lemma

Lemma posts formal proof targets, verifies submissions with Lean, and pays only through funded LemmaBountyEscrow bounties.

Open Repo Run CLI
01 Funded escrow

A reward is live only after escrow funding is confirmed on chain.

02 Lean proof

The solver submits a Submission.lean file for the exact target.

03 Binary check

Lean accepts the proof or rejects it. There is no subjective score.

04 Contract payout

The escrow contract releases funds under fixed commitment and reveal rules.

What escrow means

No funded escrow, no reward promise.

A Lemma bounty is a public registry row connected to a funded Bittensor EVM escrow bounty. Candidate targets can exist before funding, but they are not live rewards.

Registry row

Defines the theorem, target hash, policy version, toolchain, source metadata, and escrow identifiers.

Escrow contract

Holds the bounty funds and accepts commit, reveal, attestation, and finalization calls.

Local packages

The CLI builds unsigned transaction data so solvers can inspect and submit with their own wallet flow.

How proof verification works

Proof correctness is binary.

The live reward path cares about one question: does Lean accept the proof for the exact published statement under the published policy? If yes, it passes. If not, it fails. 

import Mathlib

namespace Submission

theorem target : True := by
  trivial

end Submission
