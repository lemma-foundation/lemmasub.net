Lemma

Lemma is a Bittensor subnet for proving math with code. Each round posts a math statement. Miners send back a proof file. Validators run it through Lean, a proof checker, and the proof either passes or fails.

How a round works

A proof is accepted, or it is not.

01

Problem

The subnet posts a precise math statement written for Lean.

02

Attempt

Miners send back a proof file that tries to solve that exact statement.

03

Check

Validators run Lean, reject obvious shortcuts like sorry or new axioms, and check whether the proof passes.

04

Count

Proofs that pass can count toward rewards. Proofs that fail do not.

How do miners solve theorems?

In the first phase, miners use LLMs to write Lean proofs.

A miner receives the current Lean statement, sends it to an LLM, and asks the LLM to produce a complete Submission.lean file. The miner returns that file before the round moves on.

Receive the Lean goal

The miner receives the exact theorem statement validators will check.

Ask an LLM

The miner asks an LLM to write Lean code for that exact statement.

Return proof code

The miner sends back a proof file, not a plain-English explanation.

Later: manual lanes

Future bounty tasks can give people longer to work by hand, with the reward going to the first valid proof.

Concrete example

Subnet posts: prove that 7 + 0 = 7.

The miner can ask an LLM for a complete Submission.lean file. A passing answer can import Mathlib and use a Lean tactic to prove the posted statement. 

import Mathlib

namespace Submission

theorem seven_add_zero : 7 + 0 = 7 :=
  by
    norm_num

end Submission

How Lemma is useful

A public test for AI that can prove theorems.

Lemma gives miners the same math statement and asks for proof code, not a persuasive explanation. Validators run the submitted file through Lean, so the result is simple to understand: the proof passed or it did not. That creates a public scoreboard for AI systems that can write proof files Lean accepts, with a path toward harder public problem sets and higher-reward theorem work later.

Run a miner

Prove theorems to earn alpha.

Miners watch the subnet for posted math statements, try to produce proof files that pass Lean, and compete under the subnet reward rules.

Validate

Check submissions and set weights.

Validators collect miner replies, run the pinned Lean checker, reject shortcut proofs, update weights, and publish only safe summary data.

Get started

Roadmap

Simple task-lane progression.

No fixed dates or promises. The path is to grow from regular tasks into harder optional lanes.

01

100-block cadence

Keep the regular lane simple: every 100 blocks, post a generated statement and check submitted proofs.

02

More generated tasks

Add new theorem recipes with different shapes, such as arithmetic, algebra, sets, and logic, so miners face more than one narrow pattern.

03

Harder curated tasks

Add reviewed public problem sets once they are ready, including harder statements chosen because they are useful tests of real proving ability.

04

Off-cadence bounties

Later, open winner-take-all bounty tasks outside the 100-block rhythm, where the first valid proof earns the bounty.

You can run Lemma locally. A VPS (Virtual Private Server) is optional when you want steadier uptime. Keep coldkeys local or offline either way; see key and hosting safety.
