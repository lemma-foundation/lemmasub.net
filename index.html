Formal proofs, checked by Lean

Lemma rewards proofs Lean can verify.

Lemma is a Bittensor subnet for formal proof work. The network publishes theorem targets, miners commit privately before reveal, and validators run the checker. The live question is narrow: did this committed proof file prove this theorem?

A formal proof is not a persuasive explanation. It is mathematics written in a language strict enough for software to verify. Lean accepts the proof only when every step follows from the theorem statement, imports, and allowed rules.

Source credit: target material is based on Google DeepMind's Formal Conjectures.

The loop

One target, one proof file, one check.

  1. Theorem

    A problem target defines the statement to prove and the Lean environment it belongs to.

  2. Commit

    A miner locally verifies a proof, keeps it private, and publishes a chain commitment.

  3. Reveal

    After the cutoff, the miner reveals the Lean file and nonce for validators to replay.

  4. Verify

    Lean accepts the proof or rejects it. Current-epoch verified work earns the reward.

A submission

Small file. Hard boundary.

The protocol does not ask validators to grade prose, style, or confidence. Lean either accepts the file or rejects it. Verified solvers share the earned epoch budget by commitment rank. The rest routes to the owner/burn UID instead of paying stale winners. 

import Mathlib

namespace Submission

theorem nat_two_plus_two_eq_four :
    (2 : Nat) + 2 = 4 := by
  rfl

end Submission
