Open proof-data flywheel for Lean

Train the best open mathematical prover.

Lemma is a Bittensor subnet building public Lean proof data. Miners generate machine-checkable proofs. Validators verify them with the Lean kernel. Every accepted proof becomes replayable training data for future mathematical provers.

AI can guess. Lean can check. Lemma pays for the checked data.

GitHub How it works

The loop

Generate tasks, prove them, publish the corpus.

Lemma turns proof search into public training data: generate Lean theorem tasks, search for proofs, check them with Lean, reward proof units, publish corpus rows, and use that data to train stronger provers.

  1. TasksValidators expose exact Lean theorem targets.
  2. ProofsMiners use any prover stack they want.
  3. ChecksLean decides pass or fail.
  4. DataAccepted proofs become public rows.

Why Lean

Lean turns plausible text into checked artifacts.

A model can write something that looks like a proof. Lean checks whether it really follows from the theorem statement and allowed imports. Lemma uses that mechanical boundary as the reward signal.

Corpus

The corpus is the product.

Each accepted row records the theorem, imports, toolchain, Mathlib revision, proof, source metadata, hashes, attribution, reward status, and replay information.

{
  "task_id": "lemma.sample.true_intro",
  "source_stream": "mathlib_snapshot",
  "proof_identity_source": "lean_proof_term",
  "verification": { "passed": true },
  "rewarded": true
}

Mining

Miners produce checked proof artifacts.

Miners can use retrieval, tactics, local models, hosted APIs, tree search, reinforcement learning, or human-written proof strategies. Lemma only scores the final proof artifact that passes the pinned verifier.

uv sync --extra btcli
uv run lemma setup
uv run lemma mine --once

Validation

Validators check artifacts, not claims.

Validators enforce task binding, run the Lean verifier, reject duplicates, score the first accepted unique proof for each active slot, and publish replayable corpus rows. Unsolved-slot value is not redistributed to current solvers.

uv run lemma worker --check
uv run lemma validate --once --no-set-weights

Benchmarks

Benchmarks measure progress. They are not the product.

The corpus trains provers. Benchmarks show whether those provers improve. Formal Conjectures, lean-eval, miniF2F, PutnamBench, and the IMO Grand Challenge are evaluation targets, not the v1 reward path.

Research context

AlphaProof-style ideas, open corpus mechanics.

AlphaProof-style systems showed the power of large formal curricula and Lean-grounded feedback. Lemma takes the data flywheel open: a market for verified Lean proof data, not a closed lab dataset.

  • AlphaProof and AlphaGeometry research
  • LeanNavigator state-graph generation
  • LeanConjecturer
  • LeanDojo

Lemma is independent. AlphaProof, Google DeepMind Formal Conjectures, miniF2F, PutnamBench, LeanDojo, LeanNavigator, and the IMO Grand Challenge are referenced as research context or evaluation targets only. No endorsement is implied.
