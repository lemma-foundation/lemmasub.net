Setup

Copy, paste, mine.

Choose one OpenAI-compatible provider, paste one model id, create one Bittensor hotkey, then let the Lemma CLI guide proof submission and mining.

Provider

Use any endpoint that speaks the OpenAI API shape.

Lemma only needs a base URL, API key, and model id for optional prover assistance. The command below lists models from the URL you choose, so you can copy the exact returned id into LEMMA_PROVER_MODEL.

OpenAI models API

https://api.openai.com/v1

Use your OpenAI API key, then list models at /models.

Gemini OpenAI compatibility

https://generativelanguage.googleapis.com/v1beta/openai

Use a Gemini API key with Google's OpenAI-compatible endpoint.

Chutes LLM chat docs

https://llm.chutes.ai/v1

Use a Chutes API key and list live models from the shared endpoint.

Anthropic OpenAI SDK compatibility

https://api.anthropic.com/v1

Anthropic describes this as a testing and comparison layer, not the preferred production API.

1

Install Lemma.

This downloads the CLI and installs the Bittensor command-line tools Lemma uses for wallet registration.

Terminal 
git clone https://github.com/spacetime-tao/lemma.git
cd lemma
uv sync --extra btcli

2

Paste your provider URL and key.

Start with OpenAI, or replace the base URL with Gemini, Chutes, Anthropic, or another compatible provider.

.env 
cat >> .env <<'EOF'
LEMMA_PROVER_BASE_URL=https://api.openai.com/v1
LEMMA_PROVER_API_KEY=replace_me
EOF

3

List models from that URL.

The response contains model ids. Copy one id exactly as the provider returns it.

Terminal 
source .env
curl -sS "$LEMMA_PROVER_BASE_URL/models" \
  -H "Authorization: Bearer $LEMMA_PROVER_API_KEY"

4

Paste one model id.

This tells Lemma which provider model to use when you ask the CLI for prover help.

.env 
cat >> .env <<'EOF'
LEMMA_PROVER_MODEL=copy_one_model_id_here
EOF

5

Create and register a hotkey.

Cadence mining requires a registered subnet UID. Bounty campaigns only need a hotkey signature.

btcli 
btcli wallet create --wallet.name lemma --wallet.hotkey miner
btcli subnets show --netuid 467 --network test
btcli subnets register --netuid 467 \
  --wallet.name lemma \
  --wallet.hotkey miner \
  --network test

6

Run setup, then mine.

The CLI shows the active theorem, verifies your Lean proof locally, commits privately, and starts the miner server.

Lemma CLI 
uv run lemma setup --role miner --wallet lemma --hotkey miner
uv run lemma mine
uv run lemma status

Bounties

Package a bounty proof locally.

Bounties are manual owner-emission campaigns. The package is signed by your hotkey and does not require subnet registration.

Lemma CLI 
uv run lemma mine --bounty <campaign-id> \
  --submission Submission.lean

Validators

Check before start.

Validators prebuild the Lean verifier image, configure a validator hotkey, then run the proof-checking service.

Lemma CLI 
bash scripts/prebuild_lean_image.sh
uv run lemma setup --role validator
uv run lemma validate
