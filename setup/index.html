Setup

Four commands for normal operation.

Lemma keeps wallet creation in btcli, then guides miners and validators through the checks needed before they touch the subnet.

CLI

Start with setup.

git clone https://github.com/spacetime-tao/lemma.git
cd lemma
uv sync --extra btcli
uv run lemma setup
uv run lemma mine
uv run lemma status

Wallets

Use btcli for keys.

uv sync --extra btcli
btcli wallet create --wallet.name <coldkey> --wallet.hotkey <hotkey>
btcli subnets show --netuid <netuid> --network <network>
btcli subnets register --netuid <netuid> --wallet.name <coldkey> --wallet.hotkey <hotkey> --network <network>

Miner

Commit first, reveal later.

  1. Setup

    Run uv run lemma setup --role miner and fix any wallet, chain, Docker, or registration warnings.

  2. Mine

    Run uv run lemma mine, paste a full Submission.lean, and let Lean check it locally.

  3. Serve

    After the commitment is published, Lemma starts the miner server and waits for validators to poll after reveal opens.

Validator

Check before start.

bash scripts/prebuild_lean_image.sh
uv run lemma setup --role validator
uv run lemma validate

A commitment is a public fingerprint of a private proof. It proves the miner had something before the cutoff without showing the proof text. After reveal opens, the miner sends the proof and nonce, and validators recompute the fingerprint before running Lean.
