FAQ

The basics, stated directly.

What is Lemma?

Lemma is a Bittensor subnet for formal proof work. Miners submit Lean proof files for theorem targets, and validators run Lean to determine whether each submission proves the target.

What is formal mathematics?

Formal mathematics is mathematics written with enough precision for a proof assistant to check it. Definitions, theorem statements, and proofs are expressed in a language with exact rules, so verification does not depend on interpretation or trust in the author.

What is a formal proof?

It is the proof object for a formal theorem. A traditional proof may omit steps that a trained reader can fill in. A formal proof must provide enough structure for the checker to confirm that the theorem follows from the stated assumptions.

What is a theorem target?

A target is the exact problem miners work on: a Lean theorem statement, the imports it may use, and the environment in which the proof must check. The target is the prompt, and a verifying Lean file is the answer.

Why Lean?

Lean is both a programming language and a proof checker. For Lemma, its most important property is the boundary it creates: a submission either checks in the pinned environment or it does not.

What do miners do?

Miners work on Lean theorem targets from the Lemma protocol. These are formal mathematics problems, not essay prompts or numerical quizzes. A miner can work manually, use its own tools, or run local agents; the protocol only needs the final proof file that Lean accepts.

Do miners need to submit explanations?

No. Explanations can be useful for people, research notes, or debugging, but they are not the scoring object. The live protocol rewards verified Lean proofs.

What do validators do?

Validators fetch submissions, run Lean verification, and convert verified proof results into subnet weights. The reward signal is proof correctness, not a judge score over written reasoning.

How are rewards distributed?

Validators turn verified proof results into subnet weights. If one miner verifies the active target, that miner receives the weight for the solved target. If several miners verify in the same validator batch, the weight is split equally across those miners.
