FAQ

Questions worth answering directly.

What is Lemma?

Lemma is a Bittensor subnet for formal proof work. Miners submit Lean proof files for theorem targets, and validators run Lean to determine whether each submission proves the target.

What is a formal proof?

It is a proof written in a language a computer can check. Instead of relying on a human reader to decide whether an argument is convincing, the proof checker verifies that every step is justified.

Why Lean?

Lean is both a programming language and a proof checker. For Lemma, its most important property is the boundary it creates: a submission either checks in the pinned environment or it does not.

What do miners do?

Miners produce Lean proof files. They can work manually, use their own tools, or run local agents. The protocol only needs the final proof file.

What do validators do?

Validators fetch submissions, run Lean verification, and convert verified proof results into subnet weights. The reward signal is proof correctness, not a judge score over written reasoning.

How are ties handled?

If several miners verify in the same validator batch, that batch's reward is split equally. The protocol should not prefer a lower UID just because two valid proofs landed together.

Where did the dashboard go?

The old public dashboard belonged to an earlier version of the site. Lemma is now centered on the repository, CLI, theorem targets, and verified proof submissions.
