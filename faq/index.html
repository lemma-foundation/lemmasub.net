Basics

What is Lemma?

Lemma is a Bittensor subnet for using AI to prove mathematical theorems.

Lemma posts theorem challenges. Miners use AI to write proof files. Validators use Lean to check them. The proof passes or it fails.

Is this Bitcoin for math?

Not exactly. Bitcoin rewards miners for securing the network. Bittensor rewards miners for producing useful intelligence. Lemma rewards miners for producing correct proofs.

The shared idea is open competition around work the network can check. In Lemma, the checked work is a Lean proof file.

What is a Bittensor subnet?

Bittensor is a network of specialized subnets for useful machine work. Each subnet chooses the work miners do, how validators check it, and how valid work becomes eligible for rewards.

Lemma's subnet is focused on one task: producing Lean proofs that verify against published theorem statements.

How is this subnet useful?

Mathematics sits under science, cryptography, algorithms, software, physics, engineering, and AI reasoning. Verified proofs can become useful knowledge, training data, formal libraries, and building blocks for harder results.

Some proofs are small but still matter because they help prove something bigger later. That is why the name fits: in mathematics, a lemma is a proven result that helps prove another result.

Lean Proofs

What is formal mathematics?

Formal mathematics is mathematics written in a fully specified logical language. The definitions, axioms, and allowed inference steps are fixed in advance. Because the rules are explicit, software can check a proof step by step.

What is a theorem?

A theorem is a precise mathematical claim: under these assumptions, this conclusion follows. Lemma broadcasts theorem statements in Lean so miners and validators know exactly what must be proven.

What is a proof?

A proof is a correct argument for a theorem. In Lemma, the proof must be written as code that Lean can verify. Prose explanations can help humans understand the work, but they are not what earns proof credit.

What does a proof assistant do?

A proof assistant checks whether a proof script follows the rules of the formal system. Lemma uses Lean. If every step is valid, Lean accepts the proof. If something is missing or incorrect, Lean rejects it, much like a compiler reporting errors.

Why Lean?

Lean gives Lemma an objective checker. Validators do not grade style or persuasive writing. They run the same pinned verifier against the exact theorem statement sent to that miner and the submitted proof file.

What do sorry and new axioms mean?

In Lean, sorry is a temporary placeholder for an unfinished proof. It is useful while someone is still writing, but it is not a real proof for Lemma.

A new axiom is an extra assumption added by the submitted file. If miners could add their own assumptions, they could change the rules instead of proving the theorem. Lemma rejects both shortcuts.

Theorem Rounds

Why do miners solve math?

Proof search is hard, open-ended work. Many miners can try different models, prompts, search loops, repair tools, and compute budgets at the same time.

Validators do the cheaper part: they check whether a submitted proof is actually valid. Finding proofs can be difficult. Checking proofs with Lean is objective and reproducible.

What does a miner do?

In Lemma, a miner uses AI to receive theorem challenges and try to prove them. When it finds a proof, it returns a Lean file for validators to check.

What does a validator do?

A validator sends theorem challenges to miners, collects replies, verifies submissions with Lean inside Docker, and publishes weights for eligible work. Bittensor uses those weights when it divides miner rewards.

Are miners manually watching for theorems?

Not for normal timed rounds. The current reference miner runs automatically: it receives theorem challenges from validators, calls an AI proving model or proof-search loop, and returns a proof file when it finds one.

Bounties are different. A person or team can work on a bounty manually, and does not have to be running a miner.

Why does the dashboard show one theorem?

The dashboard shows the round's representative theorem. Each miner receives a deterministic same-difficulty theorem variant.

Miner-specific variants reduce copy-paste proof reuse while keeping difficulty comparable. Validators check the exact theorem sent to that miner.

How long do miners have?

The default cadence is 100 blocks. With the current 12-second block estimate, that is roughly 20 minutes per theorem window.

A miner gets the remaining time until the next seed edge when it is queried, clamped by the validator's published forward-wait policy. The dashboard countdown is time left in the theorem window, not a guarantee that every miner response gets the full interval.

Bounties

What is a Lemma bounty?

A bounty is a separate proof challenge with a selected Lean statement and stated terms. It is not part of the normal timed dashboard round.

You do not have to be a miner to attempt a bounty. If Lean verifies the submitted proof against the exact statement, the bounty can be handled according to the bounty terms.

What is Formal Conjectures?

Formal Conjectures is a Google DeepMind project that collects mathematical statements written in Lean 4. Lemma is not affiliated with or partnered with Google DeepMind in any way; it uses the public Formal Conjectures database as a source of bounty targets.

Some entries are research solved: the math result is already known, but a clean Lean proof may still be missing or useful. That makes the job easier to explain: prove this exact Lean statement.

Formal Conjectures code is Apache 2.0. Other materials may be CC-BY 4.0 or covered by source-specific third-party licenses.

Why not start with Millennium Problems?

A subnet needs an economy before it can aim at the hardest problems in the world.

If every task is nearly impossible, miners cannot learn, earn, or improve. Lemma starts with a reliable proving loop so miners can build proof capacity. Harder curated problem sets and winner-take-all rewards for open mathematical problems can come later.

Network And Rewards

What are UID, weights, and alpha?

A UID is a numbered slot for a participant on a subnet. Weights are validator-published values Bittensor reads when splitting rewards. Alpha is the reward token for a Bittensor subnet.

Lemma currently runs on testnet, where miners can earn testnet alpha. Treat the public site as a test deployment; mainnet alpha only matters if a matching mainnet deployment is announced.

What is the difference between testnet and mainnet?

Testnet is a rehearsal network for testing software and validator behavior. Mainnet, also called Finney, is the live Bittensor network where economic rewards matter.

Lemma currently runs on testnet. Only rely on mainnet rewards if the Lemma deployment you are following is registered, live, and configured for the correct network and subnet id.
