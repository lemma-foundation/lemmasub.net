Lemma FAQ

The protocol, in ordinary words: publish a theorem, submit a Lean proof, let Lean decide.

What is Lemma?

Lemma is a Bittensor subnet for rewarding correct mathematical proofs.

Each live target is a theorem statement written for Lean. Miners submit proof files. Validators check those files with Lean. The proof passes or it fails.

What changed?

The public protocol has been simplified to winner-take-all proof verification.

The old dashboard refresh and automated proof-generation framing are retired from this site. The current public story is the direct one: first valid Lean proof wins.

What does a miner do now?

A miner generates a Lean proof out of band. That can mean an AI proving model, a local search tool, manual work, or a mix.

After that, the miner runs lemma submit to verify and store the proof, then can run lemma miner start to serve stored proofs to validators.

What does a validator do?

A validator publishes the target theorem, polls miners for stored proofs, runs the pinned Lean verifier, and rewards the first valid proof.

Validators do not grade writing style, confidence, or informal reasoning. Lean verification is the live check.

What earns credit?

A Lean proof file that proves the exact target theorem without shortcuts earns credit.

Invalid files, unfinished proofs, sorry, and new axioms do not earn credit. Proof efficiency is not the core reward axis.

Is informal reasoning useless?

No. Informal reasoning can help humans understand a result and can help miners find a proof.

It is just not the live reward object. The reward object is the Lean proof that checks.

Where did the dashboard go?

The old public dashboard JSON refresh has been retired. This repo no longer treats data/public-dashboard.json as a live surface.

Use the Lemma CLI and the GitHub docs for current protocol state.

Where do I start?

Start with the getting started guide, then read the winner-take-all notes.
